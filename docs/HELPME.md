## 시작하기

### 프로그램 실행

1. DocWatch 아이콘을 더블클릭하여 실행합니다
2. 처음 실행 시 14일 무료 체험이 자동으로 시작됩니다
3. 메인 화면이 나타나면 사용 준비가 완료된 것입니다

### 화면 구성

- **폴더 감시 탭**: 문서 폴더 모니터링 설정
- **회의 탭**: 회의 녹음 및 회의록 관리
- **설정 탭**: 프로그램 설정

### 시스템 트레이

- 창을 닫아도 프로그램은 시스템 트레이에서 계속 실행됩니다
- 트레이 아이콘을 더블클릭하면 창이 다시 나타납니다
- 트레이 아이콘 우클릭 → "종료"로 완전히 종료할 수 있습니다

## 폴더 감시

### 감시 폴더 추가하기

1. "폴더 추가" 버튼을 클릭합니다
2. 감시할 폴더를 선택합니다
3. 폴더가 목록에 추가되면 자동으로 감시가 시작됩니다

### 감시 폴더 삭제하기

1. 감시 폴더 목록에서 삭제할 폴더를 찾습니다
2. 해당 폴더 옆의 "삭제" 버튼을 클릭합니다

### 지원되는 파일 형식

| 형식 | 확장자 | 분석 기능 |
|------|--------|-----------|
| Word | .docx | 텍스트 추출, 변경 감지 |
| Excel | .xlsx | 시트/셀 분석, 변경 감지 |
| PowerPoint | .pptx | 슬라이드 분석, 변경 감지 |
| PDF | .pdf | 텍스트 추출 |
| 텍스트 | .txt | 내용 변경 감지 |

### 변경 로그 확인

- 파일이 생성, 수정, 삭제될 때마다 로그에 기록됩니다
- 최근 100건의 변경 내역을 확인할 수 있습니다
- "로그 지우기" 버튼으로 로그를 초기화할 수 있습니다

## 문서 분석

### 문서 분석하기

1. 변경 로그에서 분석하고 싶은 파일을 찾습니다
2. "분석" 버튼을 클릭합니다
3. 분석 결과가 팝업으로 나타납니다

### 분석 결과 내용

- **파일 정보**: 파일명, 크기, 수정 시간
- **변경사항**: 감지된 변경 내역
- **키워드**: 추출된 주요 키워드
- **AI 요약** (Pro): 변경사항을 자연어로 요약

### AI 요약 기능 (Pro)

- Ollama가 설치되어 있으면 자동으로 AI 요약이 생성됩니다
- 요약은 한국어로 제공됩니다
- 변경사항이 있을 때만 AI 요약이 표시됩니다

## 회의 녹음

### 녹음 시작하기

1. "회의" 탭으로 이동합니다
2. "녹음 시작" 버튼을 클릭합니다
3. 마이크 접근 권한을 허용합니다
4. 녹음 중에는 경과 시간이 표시됩니다

### 녹음 중지 및 저장

1. "녹음 중지" 버튼을 클릭합니다
2. 회의 제목을 입력합니다 (선택사항)
3. "저장" 버튼을 클릭합니다
4. 자동으로 음성 인식 및 화자 분리가 시작됩니다

### 녹음 팁

- 조용한 환경에서 녹음하면 인식률이 높아집니다
- 마이크와 적당한 거리를 유지하세요
- 여러 명이 말할 때는 차례로 말하는 것이 좋습니다
- **화자 분리 정확도를 위해**: 각 화자가 최소 5초 이상 연속 발화하면 더 정확합니다

## 회의록 관리

### 회의록 보기

1. "회의" 탭의 회의록 목록에서 항목을 선택합니다
2. "보기" 버튼을 클릭합니다
3. 원본 전사와 AI 요약이 표시됩니다

### 회의록 구성 (v2.0 업데이트)

- **원본 전사**: 음성을 그대로 텍스트로 변환한 내용
  - **화자 구분**: SPEAKER_A, SPEAKER_B 등으로 자동 구분
  - **발언량 통계**: 각 화자별 발언 비율 표시
  - **겹침 발화**: (겹침) 표시로 동시 발화 구간 표시
- **AI 요약**: 핵심 내용을 정리한 요약본
  - 화자별 주요 발언 정리
  - 액션 아이템 (담당자, 할일, 기한)
  - 결정 사항 요약
- **타임스탬프**: 각 발언의 시간 정보

### 회의록 삭제

1. 회의록 목록에서 삭제할 항목을 찾습니다
2. "삭제" 버튼을 클릭합니다
3. 확인 메시지에서 "확인"을 클릭합니다

## 화자 분리 기능

> **v2.0 신규 기능**: 회의 참석자를 자동으로 구분합니다

### 화자 분리란?

- 회의 녹음에서 "누가 말했는지"를 자동으로 구분하는 기능입니다
- 각 발언자를 SPEAKER_A, SPEAKER_B, SPEAKER_C... 형태로 표시합니다
- 발화량 기준으로 가장 많이 말한 사람이 SPEAKER_A로 지정됩니다

### 화자 분리 결과 예시

```
========================================
              회 의 록
========================================

▣ 참석자 (발언량 기준)
   - SPEAKER_A: 45.0%
   - SPEAKER_B: 35.0%
   - SPEAKER_C: 20.0%

▣ 회의 내용
----------------------------------------

[00:00] SPEAKER_A:
오늘 회의는 프로젝트 진행 상황을 점검하겠습니다.

[00:06] SPEAKER_B:
네, 현재 개발 진행률은 약 70% 정도입니다.

[00:17] SPEAKER_C:
(겹침) 테스트 관련해서 말씀드릴게요...

========================================
```

### 화자 분리 정확도 높이기

1. **명확한 발화**: 각 화자가 번갈아가며 발언할수록 정확합니다
2. **충분한 발화 시간**: 최소 0.5초 이상의 발언이 필요합니다
3. **적절한 인원**: 2~5명의 회의에서 가장 잘 작동합니다
4. **조용한 환경**: 배경 소음이 적을수록 정확합니다
5. **겹침 최소화**: 동시에 말하지 않을수록 화자 구분이 정확합니다

### 화자 이름 수정

현재 버전에서는 자동으로 생성된 SPEAKER_A, SPEAKER_B를 수동으로 수정해야 합니다.
향후 업데이트에서 화자 이름 매핑 기능이 추가될 예정입니다.

### 화자 수 제한

- 최소 2명 ~ 최대 10명까지 자동 감지
- 참석자 수가 5명을 초과하면 정확도가 낮아질 수 있습니다
- 많은 인원의 회의는 소그룹으로 나눠 녹음하는 것을 권장합니다

## 녹음 파일 관리

### 녹음 파일 목록

- "회의" 탭에서 저장된 녹음 파일 목록을 확인할 수 있습니다
- 각 파일의 이름, 크기, 녹음 날짜가 표시됩니다
- **v2.0**: 원본 파일명이 그대로 유지되어 표시됩니다

### 녹음 파일 다운로드

1. 녹음 파일 목록에서 원하는 파일을 찾습니다
2. 다운로드 아이콘을 클릭합니다
3. 파일이 다운로드 폴더에 저장됩니다

### 녹음 파일에서 회의록 생성

1. 녹음 파일 목록에서 전사할 파일을 찾습니다
2. "전사" 버튼을 클릭합니다
3. 음성 인식 + 화자 분리가 진행되고 회의록이 생성됩니다

### 녹음 파일 삭제

1. 녹음 파일 목록에서 삭제할 파일을 찾습니다
2. 휴지통 아이콘을 클릭합니다
3. 확인 메시지에서 "확인"을 클릭합니다

## 문제 해결

### Whisper가 작동하지 않을 때

**증상**: 음성 인식이 되지 않거나 "시뮬레이션 모드"로 표시됨

**해결 방법**:
1. whisper-cpp가 설치되어 있는지 확인
   ```bash
   which whisper-cli
   ```
2. 모델 파일이 존재하는지 확인
   ```bash
   ls models/ggml-small.bin
   ```
3. 모델 파일이 없으면 다운로드:
   ```bash
   curl -L -o models/ggml-small.bin \
     https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-small.bin
   ```

### Ollama가 작동하지 않을 때

**증상**: AI 요약이 생성되지 않음

**해결 방법**:
1. Ollama 서비스가 실행 중인지 확인
   ```bash
   ollama list
   ```
2. Qwen 모델이 설치되어 있는지 확인
   ```bash
   # 고품질 모델
   ollama pull qwen2.5:7b-instruct-q4_K_M

   # 또는 경량 모델
   ollama pull qwen2.5:3b
   ```
3. Ollama 서비스 재시작
   ```bash
   ollama serve
   ```

### 화자 분리가 정확하지 않을 때

**증상**: 같은 사람이 다른 화자로 분리되거나, 다른 사람이 같은 화자로 분류됨

**해결 방법**:
1. **녹음 품질 개선**
   - 마이크에 더 가까이 녹음
   - 배경 소음 제거
   - 에어컨/선풍기 끄기
2. **발화 패턴 개선**
   - 동시에 말하지 않기
   - 각 발언 사이에 잠시 쉬기
   - 최소 0.5초 이상 연속 발화
3. **참석자 수 줄이기**
   - 5명 이하로 유지
   - 많은 인원은 소그룹으로 나눠 녹음

### 녹음이 되지 않을 때

**증상**: 녹음 버튼을 눌러도 반응이 없음

**해결 방법**:
1. 브라우저/앱에서 마이크 권한을 허용했는지 확인
2. 시스템 설정에서 마이크가 활성화되어 있는지 확인
3. 다른 프로그램이 마이크를 사용 중인지 확인
4. 프로그램을 재시작

### 문서 분석이 느릴 때

**해결 방법**:
1. 감시 폴더 수를 줄여보세요
2. 대용량 파일이 많은 폴더는 피하세요
3. 네트워크 드라이브는 성능이 저하될 수 있습니다

### 포트 충돌

**증상**: "포트 4400이 사용 중" 오류

**해결 방법**:
1. 다른 DocWatch 인스턴스가 실행 중인지 확인
2. 프로그램이 자동으로 기존 프로세스를 종료하려 시도합니다
3. 수동으로 프로세스 종료:
   ```bash
   # macOS/Linux
   lsof -i :4400
   kill -9 [PID]

   # Windows
   netstat -ano | findstr 4400
   taskkill /PID [PID] /F
   ```

### 메모리 부족

**증상**: 프로그램이 느려지거나 응답하지 않음

**해결 방법**:
1. 경량 AI 모델로 변경 (Qwen 3B)
2. 다른 프로그램 종료
3. 긴 녹음은 여러 개로 분할

## FAQ

### Q: 인터넷 연결 없이 사용할 수 있나요?
**A**: 네, 모든 AI 처리가 로컬에서 이루어지므로 인터넷 연결 없이 사용 가능합니다. 단, 라이선스 활성화 시에만 온라인 연결이 필요합니다 (폐쇄망은 오프라인 키 사용).

### Q: 녹음된 데이터는 어디에 저장되나요?
**A**: 모든 데이터는 로컬 `meetings` 폴더에 저장됩니다. 외부 서버로 전송되지 않습니다.

### Q: 어떤 언어를 지원하나요?
**A**: 현재 한국어(ko) 음성 인식에 최적화되어 있습니다. Whisper는 다양한 언어를 지원하므로 설정 변경이 가능합니다.

### Q: 녹음 시간 제한이 있나요?
**A**: 소프트웨어 상의 제한은 없지만, 긴 녹음은 더 많은 저장 공간과 처리 시간이 필요합니다. 일반적으로 1시간 미만의 녹음을 권장합니다.

### Q: 동시에 여러 폴더를 감시할 수 있나요?
**A**: 네, 여러 폴더를 동시에 감시할 수 있습니다. 단, 폴더 수가 많아지면 시스템 리소스 사용량이 증가할 수 있습니다.

### Q: Trial 기간이 끝나면 기존 데이터는 어떻게 되나요?
**A**: 기존에 생성된 회의록과 분석 데이터는 그대로 유지됩니다. 다만 새로운 Pro 기능(녹음, 분석, AI 요약, 화자 분리)은 사용할 수 없습니다.

### Q: 라이선스를 다른 PC로 옮길 수 있나요?
**A**: 라이선스 관리자에게 문의하여 기기 변경 요청이 가능합니다.

### Q: 오류가 발생하면 어떻게 하나요?
**A**: 오류 메시지를 캡처하여 기술 지원팀에 문의해 주세요. 로그 파일은 앱 데이터 폴더에 저장됩니다.

### Q: 화자 분리는 몇 명까지 지원하나요?
**A**: 2명부터 10명까지 자동 감지됩니다. 5명 이하의 회의에서 가장 정확하게 작동합니다.

### Q: 화자 이름을 직접 지정할 수 있나요?
**A**: 현재 버전에서는 자동 생성된 SPEAKER_A, B, C를 수동으로 수정해야 합니다. 향후 업데이트에서 화자 이름 매핑 기능이 추가될 예정입니다.

### Q: 처리 시간은 얼마나 걸리나요?
**A**: 1시간 회의 기준 약 20~40분 소요됩니다 (CPU 성능에 따라 다름). 세부 시간:
- 오디오 변환: ~10초
- 음성 인식: ~15-30분
- 화자 분리: ~2-5분
- AI 요약: ~2-5분
